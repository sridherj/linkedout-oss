# Shared Context: Phase 08 — Skill System & Distribution

## Goal
Build the cross-platform skill infrastructure. Skills are the user-facing interface — the primary way users interact with LinkedOut. Users never interact with CLI commands directly; they invoke `/linkedout` and the skill translates natural language to CLI calls and DB queries. This phase delivers the template system, host configs, all 6 skill definitions, routing files, generation script, and dev workflow.

### Template System
- Templates use `{{VARIABLE_NAME}}` syntax for variable substitution
- Conditional blocks: `{{#if claude}}...{{/if}}`
- Variables resolved from host config + global context
- Frontmatter processed per host config rules (denylist/allowlist mode, field stripping, description limits)
- Path rewrites applied per host (e.g., `.claude/skills/` → `.agents/skills/` for Codex)
- Tool name rewrites where needed
- Generated files get header: `<!-- Generated by bin/generate-skills — do not edit -->`
- Python-based (no Node.js), using `string.Template` or minimal Jinja2 subset

### Template Variables
`{{SKILL_NAME}}`, `{{SKILL_DESCRIPTION}}`, `{{VERSION}}`, `{{DATA_DIR}}`, `{{CONFIG_DIR}}`, `{{CLI_PREFIX}}`, `{{HOST_NAME}}`, `{{AGENT_CONTEXT_PATH}}`, `{{DB_SCHEMA_SNIPPET}}`

### Host Config Schema (YAML)
```yaml
name: claude
display_name: Claude Code
skill_install_path: ~/.claude/skills/linkedout
local_skill_path: .claude/skills/linkedout
frontmatter:
  mode: denylist          # denylist = strip listed; allowlist = only keep listed
  strip_fields: []
  description_limit: null
path_rewrites: []         # [{from: "...", to: "..."}]
tool_rewrites: {}         # {AskUserQuestion: input}
```

## Codebase Conventions
- **Build system:** `uv run` for Python commands. Dependencies in `backend/requirements.txt`.
- **ORM:** SQLAlchemy 2.0 with `Mapped[]` type annotations. Alembic for migrations.
- **Entities:** Domain entities in `backend/src/linkedout/<domain>/entities/`. Base class: `BaseEntity`.
- **Services:** Business logic in `backend/src/linkedout/<domain>/services/`.
- **CLI:** Click commands in `backend/src/dev_tools/`. Rewired to `linkedout` namespace.
- **Logging:** loguru via `get_logger()`. Bind `component` and `operation` fields.
- **Config:** pydantic-settings via `backend/src/shared/config/config.py`. Env vars override YAML. Prefix: `LINKEDOUT_`.
- **DB sessions:** `db_session_manager.get_session(DbSessionType.READ|WRITE, app_user_id=...)` context manager.
- **Tests:** pytest in `backend/tests/`. Unit tests mock external APIs. Integration tests use real DB.
- **pgvector:** `pgvector.sqlalchemy.Vector` type for embedding columns. HNSW indexes for search.
- **System user:** `SYSTEM_USER_ID` from `dev_tools.db.fixed_data` for CLI operations.

## Key DB Tables (for schema-reference.md)

Core tables used by the `/linkedout` skill:
- `connection` — LinkedIn connections
- `crawled_profile` — Enriched profile data with embeddings
- `experience` — Work experience entries
- `education` — Education history
- `profile_skill` — Skills per profile
- `company` — Company records
- `company_alias` — Alternative company names
- `role_alias` — Alternative role/title names
- `funding_round` — Company funding data
- `startup_tracking` — Startup monitoring
- `growth_signal` — Growth indicators

All queries must filter by tenant/BU/user IDs (RLS context from `agent-context.env`).

## Pre-Existing Decisions (Resolved 2026-04-07, SJ)

1. **Generated files in git:** Yes — track in git. Users can install skills directly from clone without running generation. CI check `bin/generate-skills --check` catches drift.
2. **Copilot skill format:** Implement based on current docs, flag as "beta." Near-zero marginal effort.
3. **Schema reference:** Auto-generate from existing `build_schema_context()`. Create `bin/generate-schema-ref` as a thin wrapper (~10 lines) that calls `build_schema_context()` and writes to `skills/linkedout/schema-reference.md`. Include in `bin/generate-skills` pipeline.
4. **Skill prefix:** No toggle. `/linkedout` is the only prefix. No configurability.
5. **Agent tool in skills:** Core `/linkedout` skill SHOULD use Agent tool for parallel query execution. Simple utility skills stay sequential.
6. **Extension setup skill:** Do NOT create a stub in Phase 8. Created in Phase 12 after UX design gate.
