# Sub-Phase 3b: Embedding Similarity Signal

**Goal:** linkedin-ai-production
**Phase:** 3 — Affinity V2 Enhancements
**Depends on:** SP-1 (Schema Migration)
**Estimated effort:** 1 session (~2-3h)
**Source plan section:** Sub-phase 3b

---

## Objective

Compute cosine similarity between the user's own profile embedding (1536-dim, text-embedding-3-small) and each connection's profile embedding. Store result in `affinity_embedding_similarity`. Returns 0.0 when either embedding is missing.

## Context

`crawled_profile.embedding` stores 1536-dim vectors generated by OpenAI text-embedding-3-small. The user's own profile is linked via `app_user.own_crawled_profile_id`. pgvector's `<=>` operator enables efficient DB-side cosine distance computation, avoiding 144MB of float data transfer to Python.

## Tasks

1. **Define `_compute_embedding_similarity()` pure function** in `affinity_scorer.py`:
   - Input: two optional embedding vectors (`list[float] | None`)
   - If either is None: return 0.0
   - Compute cosine similarity: `dot(a, b) / (norm(a) * norm(b))`
   - Clamp to [0.0, 1.0] (negative cosine = anti-correlated; treat as 0)
   - Use numpy for efficiency (already a dependency via pgvector)

2. **Add `_get_user_embedding()` method** to `AffinityScorer`:
   - Query user's own `crawled_profile.embedding` via `app_user.own_crawled_profile_id`
   - Return the embedding vector or None

3. **Add `_batch_fetch_embedding_similarities()` method** — preferred DB-side approach using pgvector:
   ```sql
   SELECT c.id, 1 - (cp.embedding <=> :user_embedding) AS similarity
   FROM connection c
   JOIN crawled_profile cp ON c.crawled_profile_id = cp.id
   WHERE c.app_user_id = :app_user_id
     AND cp.embedding IS NOT NULL
   ```
   Returns `dict[str, float]` keyed by connection_id. Connections without embeddings get 0.0 (excluded by WHERE, handled in Python).

4. **Wire into `compute_for_user()` and `compute_for_connection()`**:
   - If user embedding is None, skip (all connections get 0.0)
   - Otherwise, use batch query results to populate `conn.affinity_embedding_similarity`

5. **Unit tests** for the pure function (Python-side cosine similarity):
   - `_compute_embedding_similarity(None, embedding)` returns 0.0
   - `_compute_embedding_similarity(embedding, None)` returns 0.0
   - Known hand-computed cosine similarity with small vectors
   - Identical vectors return 1.0
   - Orthogonal vectors return 0.0
   - Result bounded 0-1

6. **Integration tests:**
   - User with `own_crawled_profile_id` and profile with embedding — connections with embeddings get nonzero similarity
   - User without `own_crawled_profile_id` — all connections get 0.0

## Design Notes

- **DB-side vs Python-side:** Use pgvector `<=>` operator for batch computation. Avoids transferring 144MB of vector data. Falls back to Python-side pure function for `compute_for_connection()` single-connection case if needed.
- **SQLite compatibility:** The `<=>` operator is PostgreSQL-only. Unit tests use the Python-side pure function. DB-side computation tested only in integration tests.
- **Performance:** 24K sequential cosine comparisons against one vector takes <1s with pgvector. No index needed for batch recomputation (computing ALL pairs, not nearest-neighbor search).
- **Dimension:** DB has 1536-dim vectors (text-embedding-3-small), not 768-dim nomic. Spec already corrected.
- **Missing embeddings:** If user's own profile has no embedding, all connections get 0.0. Expected.
