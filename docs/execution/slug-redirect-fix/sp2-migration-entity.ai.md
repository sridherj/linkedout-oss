# SP2: Migration + Entity — Add `previous_linkedin_url`

**Depends on:** Nothing (can run in parallel with SP1)
**Produces:** New column + index on `crawled_profile` table, entity updated
**Estimated scope:** ~15 lines entity change + Alembic migration

## Overview

Add a `previous_linkedin_url` column to `CrawledProfileEntity` to preserve the original CSV-imported URL when enrichment detects a LinkedIn slug redirect. This column is needed by SP3 (URL update) and SP4 (import dedup).

## Changes

### 1. Update `CrawledProfileEntity`

**File:** `backend/src/linkedout/crawled_profile/entities/crawled_profile_entity.py`

Add the new column after `linkedin_url` (line 17):

```python
previous_linkedin_url: Mapped[Optional[str]] = mapped_column(
    String(500), nullable=True, comment='Previous LinkedIn URL before redirect canonicalization'
)
```

Add the index to `__table_args__` (after `ix_cp_linkedin_url`):

```python
Index('ix_cp_prev_linkedin_url', 'previous_linkedin_url'),
```

### 2. Create Alembic migration

Generate a new migration:

```bash
cd /data/workspace/linkedout-oss/backend && uv run alembic revision --autogenerate -m "add previous_linkedin_url to crawled_profile"
```

The migration should contain:
- `op.add_column('crawled_profile', sa.Column('previous_linkedin_url', sa.String(500), nullable=True, comment='...'))`
- `op.create_index('ix_cp_prev_linkedin_url', 'crawled_profile', ['previous_linkedin_url'])`

And the downgrade:
- `op.drop_index('ix_cp_prev_linkedin_url', 'crawled_profile')`
- `op.drop_column('crawled_profile', 'previous_linkedin_url')`

Review the autogenerated migration to ensure it only contains the expected changes. If it picks up unrelated drift, trim it.

### 3. Apply migration

```bash
cd /data/workspace/linkedout-oss/backend && uv run alembic upgrade head
```
