## 4. Encoding Errors

Total encoding errors encountered: **2**

Both files contain binary/corrupted data embedded in JSON strings (likely corrupted
profile picture URLs or base64 image data). Each binary blob spans ~8-10MB.
The parser recovers by scanning forward for the next valid profile object.
An estimated 1-2 profiles per blob are lost in the corrupted regions.

## 8. Key Observations for Import Pipeline

1. **Month format**: Experience/education `startDate.month` is text ('Feb', 'Aug') not integer. Needs parsing.
2. **Company extraction**: Companies must be extracted from both `experience` and `currentPosition` arrays. `companyUniversalName` is only in experience, not currentPosition.
3. **Skills**: Apify `skills` have `name` and `positions` (list of strings). No endorsement count available.
4. **Deduplication**: Cross-file overlap should be checked. Within-file duplicates exist (same linkedinUrl appears multiple times).
5. **Binary corruption**: ~8-10MB binary blobs appear in both files, likely from corrupted image downloads. Import must handle or skip these.
6. **Raw storage**: Store entire profile/experience/education objects as JSONB in `raw_*` columns for future field extraction.
7. **Location parsing**: Apify pre-parses location into city/state/country/countryCode in `location.parsed`. Fall back to `location.linkedinText` if parsed is null.
8. **Current position**: The `currentPosition` array is separate from `experience`. Both should be checked for current role.
